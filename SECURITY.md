# Security Policy

## Supported Versions

Security fixes are provided for the latest stable release line only.

| Version | Supported |
| --- | --- |
| latest stable release line | Yes |
| older stable releases | No |
| `beta` / `alpha` / `snapshot` | No |

## Reporting a Vulnerability

If you discover a security vulnerability in ServerHarbor, do not open a public issue before contacting us privately.

Please report sensitive issues through one of the following channels:

- GitHub Security Advisories / private vulnerability report
- Email: `mail@sunnyhmz.top`

When possible, include:

- affected version(s)
- vulnerability type and impact
- reproduction steps or a minimal proof of concept
- suggested remediation details, if available

## Scope

This policy mainly covers:

- bootstrap and hardening logic that changes system configuration
- peer probing and remote synchronization behavior
- backup, integrity, and scheduled execution logic
- repository-managed scripts and configuration defaults
